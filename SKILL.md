# Project Athena — AI-Generated Video Detection Knowledge Base

## Overview

Project Athena is an app that, with user permission, analyzes YouTube videos to determine if they are AI-generated or real. This skill contains the accumulated research and detection methodology.

---

## Detection Approaches

AI-generated video detection techniques fall into **four main categories**:

### 1. Watermarking & Provenance (Cryptographic)

**The most reliable method when available — but requires cooperation from the AI generator.**

#### SynthID (Google DeepMind)
- Invisible watermark embedded at pixel level into AI-generated video
- Works for images, video, audio, and text
- Detectable via Google's verification tools (e.g., Gemini app can ask "Was this generated using Google AI?")
- Limitation: Only works for content generated by participating Google models ( Imagen, Veo )
- Detection is robust against compression/resizing but can be stripped by sophisticated attackers

#### C2PA / Content Credentials
- Open standard for content provenance (developed by Adobe, Arm, BBC, Intel, Microsoft, Google)
- Functions like a "nutrition label" for digital content
- Stores metadata about creation tool, editing history, AI generation info
- Embedded in file metadata (not visible to viewer)
- Platforms like TikTok, Meta, YouTube can read C2PA and auto-label content
- Limitation: Can be stripped; not all AI tools embed C2PA; self-reported by creator

#### YouTube's Self-Labeling System
- YouTube requires creators to disclose "materially altered or synthetically generated" content
- AI indicator icon appears on videos (circle with "AI" label, bottom-left on mobile)
- Based on creator self-disclosure + some automated detection
- Limitation: Self-labeling is voluntary; many creators don't disclose

### 2. Visual / Perceptual Analysis (Artifacts)

**These look for statistical and visual artifacts left by AI generation models.**

#### Facial Inconsistencies
- **Blinking abnormalities**: AI often fails to generate natural blinking patterns (too frequent, unnatural motion)
- **Mouth-lip sync issues**: Mouth movements don't perfectly match audio; teeth look uncanny
- **Eye gaze direction**: Pupils may not track naturally; eyes look "off"
- **Facial landmark displacements**: Position of nose/mouth/eyes relative to face contour inconsistent
- **Skin texture smoothing**: AI tends to produce unnaturally smooth/porcelain-like skin
- **Facial warping artifacts**: Edges of face show warping or blur

#### Temporal Artifacts (Frame-by-Frame Analysis)
- **Frame consistency**: Objects/backgrounds change incorrectly across frames
- **Motion anomalies**: Hair/clothing/background move unnaturally; physics violations
- **Pixel-wise temporal frequency**: Fake videos show specific frequency patterns across frames that differ from real footage
- **Stitching artifacts**: When multiple AI clips are joined (~every 8 seconds), visual discontinuities appear

#### Visual Quality Indicators
- **Resolution & compression trick**: AI videos often intentionally degraded to hide artifacts (low-res + heavy compression)
- **Uncanny valley in details**: Slight wrongness in hands (extra fingers), text in background, logo placement
- **Background inconsistencies**: Objects move in impossible ways; shadows don't match lighting

#### Spatial Frequency Analysis
- AI-generated images/videos have distinct noise patterns in spatial frequency domain
- Deep learning models are trained on real data and leave statistical fingerprints

### 3. Deep Learning-Based Detection

**Trained classifiers that detect AI-generated content based on learned patterns.**

#### State-of-the-Art Models (2024-2026)
- **3DCNNDetection** — Uses appearance + motion + geometry feature fusion (arXiv:2406.19568)
- **DeMamba** — Mamba-based modeling of spatial-temporal artifacts on million-scale video
- **UNITE** — Universal synthetic video detector for face/background manipulations AND fully AI-generated content
- **BusterX / BusterX++** — MLLM-powered detection with reasoning/explanation
- **VIDGUARD-R1** — Uses GRPO with specialized reward models for temporal artifacts and generation complexity
- **STALL** — Training-free zero-shot detection using spatio-temporal likelihoods from pretrained embeddings
- **NSG-VD** — Physics-driven spatio-temporal modeling using normalized spatio-temporal gradient statistics
- **D3** — Training-free using second-order central difference features
- **Spatiotemporal Inconsistency Learning (STIL)** — Models temporal artifacts across frames

#### Detection Pipeline (Typical)
1. Face detection and tracking through video frames
2. Extract spatial features per frame (CNN backbone)
3. Model temporal dynamics across frames (LSTM/Transformer)
4. Classify as real vs AI-generated

### 4. Metadata & Contextual Analysis

**Using external signals to assess likelihood of AI generation.**

- **Upload timing patterns**: Channels posting at unusual intervals
- **Description/metadata**: Check for AI tool mentions, unusual descriptions
- **Channel characteristics**: High view/sub ratio, generic content, templated descriptions
- **Video length**: AI videos tend to be short (6-10 seconds before artifacts accumulate)
- **Quality patterns**: Deliberately low quality to hide generation artifacts

## For Project Athena Implementation

### Recommended Multi-Layer Detection Strategy

```
Layer 1: Provenance Check (fast, most reliable)
├─ Check C2PA metadata in video file
├─ Check YouTube AI label indicator
└─ Check SynthID watermark detection

Layer 2: Visual Artifact Analysis (medium cost)
├─ Face landmark consistency
├─ Temporal motion analysis
├─ Spatial frequency anomaly detection
└─ Blinking/mouth sync checks

Layer 3: Deep Learning Classifier (higher cost)
├─ Pre-trained model (e.g., BusterX, DeMamba)
└─ Ensemble voting for confidence

Layer 4: Contextual Signals (optional)
├─ Channel behavior patterns
└─ Video metadata analysis
```

### Key Challenges
1. **Evolvable AI**: Detection methods become obsolete as AI improves — continuous retraining needed
2. **Compressed video**: YouTube re-encodes videos, stripping metadata and degrading watermark detection
3. **Low-quality hides artifacts**: Attackers deliberately degrade quality to evade detection
4. **Short clips**: Many AI videos are <10 seconds, limiting temporal analysis
5. **False positives**: Real videos shot in certain conditions may trigger false alarms

### YouTube API Considerations
- YouTube provides AI content labels on some videos (self-reported by creators)
- Video metadata can be read via YouTube Data API v3
- No public API for deepfake/AI detection — must be implemented separately
- Video download requires handling DASH manifests and re-encoding

### A. Facial Inconsistencies Detection

**What to detect:**
- Unnatural blinking patterns (AI blinks too frequently or not at natural intervals)
- Mouth-lip sync issues (teeth movement inconsistent with speech)
- Eye gaze direction (pupils don't track naturally)
- Facial landmark displacement (nose/mouth/eyes misaligned relative to face contour)
- Uncanny skin texture (AI tends to over-smooth skin)

**Pipeline:**
```
Video Frame → Face Detection → Landmark Extraction → Anomaly Scoring → Decision
```

#### Step 2: Blinking Pattern Analysis

Real humans blink 10-20 times per minute, with mean duration ~100-400ms. AI-generated videos often have:
- Too frequent blinks (some models try to mimic but fail)
- Unnatural eyelid shape during blink
- No micro-blinks (real humans do these)
- Synchronization issues (both eyes don't move together)

```python
class BlinkDetector:
    def __init__(self, fps=30):
        self.fps = fps
        self.eye_openness_history = []
        self.blink_events = []
        
    def compute_openness(self, landmarks):
        """Compute eye openness from landmark distances"""
        # Vertical distance between upper and lower eyelid
        left_openness = self.landmark_distance(landmarks['left_upper'], 
                                              landmarks['left_lower'])
        right_openness = self.landmark_distance(landmarks['right_upper'],
                                               landmarks['right_lower'])
        return (left_openness + right_openness) / 2
    
    def detect_blink(self, frame_landmarks, timestamp):
        """Detect a blink event and classify as natural/anomalous"""
        openness = self.compute_openness(frame_landmarks)
        self.eye_openness_history.append((timestamp, openness))
        
        # Sliding window for blink detection
        if len(self.eye_openness_history) > self.fps:  # 1 second window
            window = self.eye_openness_history[-self.fps:]
            
            # Blink detection: openness drops below threshold
            if self.is_blink_event(window):
                blink_duration = self.get_blink_duration(window)
                blink_count = self.count_blinks_in_window(window)
                
                # Classify
                anomaly_score = self.classify_blink_anomaly(
                    blink_count, blink_duration, window
                )
                return anomaly_score
        
        return 0.0  # No anomaly detected
    
    def classify_blink_anomaly(self, blink_count, duration, window):
        """Score how anomalous this blinking pattern is (0-1)"""
        score = 0.0
        
        # Too many blinks (>20/min = anomaly)
        if blink_count > 20:
            score += 0.4 * (blink_count - 20) / 10
        
        # Unusual duration (normal: 100-400ms)
        if duration < 80 or duration > 500:
            score += 0.3
        
        # Asynchronous eye movement (both eyes should move together)
        if not self.eyes_synchronized(window):
            score += 0.3
        
        return min(score, 1.0)
```

#### Step 3: Mouth/Lip Sync Analysis

AI often struggles to perfectly sync mouth movements with audio, especially with teeth and tongue.

```python
class LipSyncDetector:
    def __init__(self):
        self.mouth_landmarks = None  # Track mouth region landmarks
        
    def extract_mouth_features(self, landmarks):
        """Extract mouth shape metrics"""
        # Key mouth landmarks (MediaPipe indices)
        upper_lip = landmarks[13]
        lower_lip = landmarks[14]
        mouth_left = landmarks[61]
        mouth_right = landmarks[291]
        
        # Mouth open width and height
        width = euclidean_distance(mouth_left, mouth_right)
        height = euclidean_distance(upper_lip, lower_lip)
        
        # Lip corner positions (for smile detection)
        left_corner = landmarks[61]
        right_corner = landmarks[291]
        
        return {
            'open_height': height,
            'open_width': width,
            'aspect_ratio': height / (width + 1e-6),
            'corners': (left_corner, right_corner)
        }
    
    def detect_sync_anomaly(self, video_frames, audio_data):
        """Detect mouth-audio desync"""
        mouth_states = []
        for frame, ts in video_frames:
            features = self.extract_mouth_features(detect_face_landmarks(frame))
            mouth_states.append((ts, features))
        
        # Compare mouth movement envelope with audio envelope
        mouth_env = self.compute_mouth_envelope(mouth_states)
        audio_env = self.compute_audio_envelope(audio_data)
        
        # Cross-correlation to find offset
        offset = self.find_offset(mouth_env, audio_env)
        
        # If offset is significant (>/< threshold), flag as suspicious
        if abs(offset) > 0.1:  # 100ms threshold
            return self.compute_anomaly_score(offset)
        
        return 0.0
```

### B. Temporal Artifacts Detection

**What to detect:**
- Frame-to-frame inconsistency (objects change incorrectly)
- Motion anomalies (hair/clothing move unnaturally, physics violations)
- Stitching artifacts (~every 8 seconds when AI clips are joined)
- Pixel-wise temporal frequency anomalies

**Key Insight:** Real videos follow physics and have natural motion continuity. AI videos often show subtle discontinuities that are hard for humans to spot but statistically detectable.

#### Step 1: Optical Flow Analysis

Optical flow tracks pixel motion between consecutive frames. AI-generated videos often show unnatural motion patterns.

```python
class TemporalAnalyzer:
    def __init__(self, fps=30):
        self.fps = fps
        self.flow_history = []
        
    def compute_optical_flow(self, prev_frame, curr_frame):
        """Compute dense optical flow between frames"""
        prev_gray = cv2.cvtColor(prev_frame, cv2.COLOR_BGR2GRAY)
        curr_gray = cv2.cvtColor(curr_frame, cv2.COLOR_BGR2GRAY)
        
        # Farneback algorithm (good balance speed/accuracy)
        flow = cv2.calcOpticalFlowFarneback(
            prev_gray, curr_gray,
            None,
            pyr_scale=0.5, levels=3, winsize=15,
            iterations=3, poly_n=5, poly_sigma=1.2,
            flags=0
        )
        
        return flow  # (h, w, 2) - magnitude in x and y
    
    def detect_motion_anomaly(self, flow, prev_landmarks, curr_landmarks):
        """Check if facial landmark motion follows natural physics"""
        # Compute expected motion from previous position
        expected_positions = self.predict_next_positions(prev_landmarks)
        
        # Compute actual motion
        actual_motion = curr_landmarks - prev_landmarks
        
        # Motion residuals (where actual deviates from expected)
        motion_residual = np.abs(actual_motion - expected_positions)
        
        # Flag anomalies where residual is too large
        anomaly_mask = motion_residual > MOTION_THRESHOLD
        
        # Score based on percentage of landmarks with anomaly
        anomaly_score = np.mean(anomaly_mask)
        
        return anomaly_score
    
    def detect_stitching_artifacts(self, frames, fps=30):
        """
        Detect cuts/stitches in AI-generated video.
        AI videos are often拼接 (stitched) from multiple clips (~8 sec each).
        """
        stitching_scores = []
        
        for i in range(len(frames) - 1):
            # Compute histogram difference between frames
            hist_prev = cv2.calcHist([frames[i]], [0], None, [256], [0, 256])
            hist_curr = cv2.calcHist([frames[i+1]], [0], None, [256], [0, 256])
            
            hist_diff = np.sum(np.abs(hist_prev - hist_curr))
            
            # Also check motion coherence across the boundary
            flow = self.compute_optical_flow(frames[i], frames[i+1])
            flow_magnitude = np.linalg.norm(flow, axis=2)
            
            # Sudden change in flow pattern = potential cut/stitch
            stitching_scores.append({
                'frame_index': i,
                'hist_diff': hist_diff,
                'flow_change': np.std(flow_magnitude)
            })
        
        # Flag frames with high stitching probability
        suspicious_boundaries = [
            s for s in stitching_scores 
            if s['hist_diff'] > HIST_DIFF_THRESHOLD 
            and s['flow_change'] > FLOW_CHANGE_THRESHOLD
        ]
        
        return suspicious_boundaries
```

#### Step 2: Temporal Frequency Analysis

AI-generated videos leave statistical traces in the frequency domain that differ from real videos.

```python
class TemporalFrequencyAnalyzer:
    """
    Based on research: 'Pixel-wise Temporal Frequency-based Deepfake Video Detection'
    arXiv:2507.02398
    """
    
    def detect(self, video_frames):
        """
        Analyze pixel-wise temporal frequency patterns.
        Fake videos show distinct frequency signatures across frames.
        """
        h, w = video_frames[0].shape[:2]
        
        # For each pixel, collect temporal series
        # Use FFT to find dominant frequencies
        
        all_fft_features = []
        
        for y in range(0, h, SUBSAMPLE):
            for x in range(0, w, SUBSAMPLE):
                # Get pixel value across all frames
                temporal_series = [frame[y, x, 0] for frame in video_frames]
                
                # Compute FFT
                fft = np.fft.fft(temporal_series)
                magnitude = np.abs(fft)
                
                # Look for abnormal frequency peaks
                # Real videos: smooth, gradual frequency distribution
                # AI videos: specific frequency spikes at certain bands
                
                feature = self.extract_frequency_signature(magnitude)
                all_fft_features.append(feature)
        
        # Aggregate and classify
        avg_signature = np.mean(all_fft_features, axis=0)
        
        # Detect if signature matches AI-generated pattern
        anomaly_score = self.classify_frequency_anomaly(avg_signature)
        
        return anomaly_score
    
    def extract_frequency_signature(self, magnitude):
        """Extract frequency band energies"""
        n = len(magnitude)
        # Divide into frequency bands
        low_band = np.mean(magnitude[1:n//4])
        mid_band = np.mean(magnitude[n//4:n//2])
        high_band = np.mean(magnitude[n//2:3*n//4])
        
        return [low_band, mid_band, high_band]
```

### C. Spatial Frequency Analysis

**What to detect:**
- AI-specific noise patterns in frequency domain
- Face warping artifacts at edges
- Unnatural texture coherence

```python
class SpatialFrequencyAnalyzer:
    """
    Based on: 'Exposing DeepFake Videos By Detecting Face Warping Artifacts'
    CVPR 2019 - Lyu et al.
    """
    
    def detect_face_warping(self, face_roi):
        """
        AI-generated faces show warping artifacts at face boundaries.
        The face synthesis process introduces subtle distortion near edges.
        """
        # Upsample the face region
        upsampled = cv2.resize(face_roi, (2*w, 2*h), interpolation=cv2.INTER_CUBIC)
        
        # Downsample back to original size
        downsampled = cv2.resize(upsampled, (w, h), interpolation=cv2.INTER_AREA)
        
        # Compute residual (AI-generated faces show warping patterns here)
        residual = np.abs(face_roi.astype(float) - downsampled.astype(float))
        
        # Compute edge mask (where warping is most visible)
        edge_mask = self.get_edge_mask(face_roi)
        
        # Score based on residual energy at edges
        edge_residual = residual * edge_mask
        anomaly_score = np.mean(edge_residual)
        
        return anomaly_score
    
    def detect_frequency_domain_anomaly(self, frame):
        """
        Use DCT (Discrete Cosine Transform) to analyze frequency components.
        AI-generated images have distinct high-frequency patterns.
        """
        # Convert to grayscale
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        # 2D DCT
        dct = cv2.dct(gray.astype(float))
        
        # Analyze frequency distribution
        # AI-generated images often have unusual HF energy distribution
        h, w = dct.shape
        
        # Compute energy in different frequency bands
        low_freq_energy = np.mean(np.abs(dct[:h//4, :w//4]))
        mid_freq_energy = np.mean(np.abs(dct[h//4:h//2, w//4:w//2]))
        high_freq_energy = np.mean(np.abs(dct[h//2:, w//2:]))
        
        # Ratio-based anomaly detection
        if high_freq_energy / (low_freq_energy + 1e-6) > THRESHOLD:
            return 0.7  # Suspicious
        
        return 0.0  # Normal
```

### D. Complete Visual Artifact Pipeline

```python
class VisualArtifactDetector:
    """
    Complete Layer 2 detector combining all visual artifact analysis.
    Runs on each video frame and produces per-frame and video-level scores.
    """
    
    def __init__(self):
        self.blink_detector = BlinkDetector()
        self.lip_sync_detector = LipSyncDetector()
        self.skin_analyzer = SkinTextureAnalyzer()
        self.temporal_analyzer = TemporalAnalyzer()
        self.spatial_analyzer = SpatialFrequencyAnalyzer()
        
        # Thresholds (tune on validation set)
        self.BLINK_ANOMALY_THRESHOLD = 0.6
        self.SKIN_ANOMALY_THRESHOLD = 0.5
        self.TEMPORAL_ANOMALY_THRESHOLD = 0.5
        
    def analyze_frame(self, frame, timestamp_ms, prev_frame=None):
        """Analyze a single frame for visual artifacts"""
        scores = {}
        
        # 1. Face detection
        faces = self.detect_faces(frame)
        if len(faces) == 0:
            return None  # No face detected
        
        face = faces[0]  # Process main face
        landmarks = self.extract_landmarks(face)
        
        # 2. Blink analysis
        scores['blink'] = self.blink_detector.detect_blink(landmarks, timestamp_ms)
        
        # 3. Lip sync analysis (needs sequence of frames + audio)
        # scores['lip_sync'] = self.lip_sync_detector.analyze(...)  # Requires audio
        
        # 4. Skin texture
        face_roi = self.get_face_roi(frame, face)
        scores['skin_texture'] = self.skin_analyzer.analyze(face_roi)
        
        # 5. Temporal analysis (if previous frame available)
        if prev_frame is not None:
            scores['temporal'] = self.temporal_analyzer.detect_motion_anomaly(
                prev_frame, frame, landmarks
            )
        
        # 6. Spatial frequency
        scores['spatial_freq'] = self.spatial_analyzer.detect_frequency_domain_anomaly(frame)
        
        return scores
    
    def analyze_video(self, video_path, max_frames=None):
        """Analyze entire video and produce final verdict"""
        cap = cv2.VideoCapture(video_path)
        fps = cap.get(cv2.CAP_PROP_FPS)
        
        all_scores = []
        prev_frame = None
        frame_idx = 0
        
        while cap.isOpened():
            ret, frame = cap.read()
            if not ret:
                break
            
            timestamp_ms = int(frame_idx * 1000 / fps)
            
            frame_scores = self.analyze_frame(frame, timestamp_ms, prev_frame)
            if frame_scores:
                all_scores.append(frame_scores)
            
            prev_frame = frame
            frame_idx += 1
            
            if max_frames and frame_idx >= max_frames:
                break
        
        cap.release()
        
        # Aggregate scores
        final_scores = self.aggregate_scores(all_scores)
        
        return final_scores
    
    def aggregate_scores(self, frame_scores):
        """
        Aggregate per-frame scores into video-level verdict.
        Uses temporal smoothing to reduce false positives.
        """
        df = pd.DataFrame(frame_scores)
        
        # Compute statistics per metric
        results = {}
        for col in df.columns:
            results[f'{col}_mean'] = df[col].mean()
            results[f'{col}_std'] = df[col].std()
            results[f'{col}_max'] = df[col].max()
            results[f'{col}_p95'] = np.percentile(df[col], 95)
        
        # Weighted combination
        # Blink and temporal anomalies are more reliable than skin texture
        weights = {
            'blink': 0.25,
            'skin_texture': 0.15,
            'temporal': 0.30,
            'spatial_freq': 0.20,
            'lip_sync': 0.10
        }
        
        final_score = 0.0
        for key, weight in weights.items():
            if f'{key}_p95' in results:
                final_score += weight * results[f'{key}_p95']
        
        # Decision threshold
        is_ai_generated = final_score > 0.5
        
        return {
            'ai_score': final_score,
            'is_ai_generated': is_ai_generated,
            'confidence': abs(final_score - 0.5) * 2,  # 0-1 confidence
            'detailed_scores': results
        }
```

### F. Practical Implementation Considerations

1. **Frame Sampling Rate**: Full analysis is expensive. Consider analyzing every 5th frame for initial screening, full analysis on suspicious videos.

2. **Face Detection Failures**: Videos with no detected face cannot use facial artifact methods. Fall back to general temporal/spatial methods.

3. **Video Quality Impact**: Heavy compression (YouTube re-encoding) degrades artifact detection. Low-quality videos are harder to analyze.

4. **Multi-Face Videos**: Currently analyzing the largest/most central face. Could extend to all detected faces for ensemble scoring.

5. **Real-World False Positives**: 
   - Low-light videos can trigger skin texture anomalies
   - Animation/cartoons will score high on many metrics (not real footage)
   - Old videos with heavy compression may show false temporal anomalies

6. **Combining with Layer 3 (Deep Learning)**: Visual artifact analysis is great for explainability but deep learning classifiers (Layer 3) often have higher accuracy. Use visual analysis for interpretability, DL for final verdict.

### G. Anatomical Inconsistencies (Hands, Body)

**Why AI struggles:** Hands are a small fraction of training data and have complex articulation. AI frequently generates wrong number of fingers, unnatural poses, or merging of hands with objects.

#### Hand/Finger Detection
```python
class HandAnomalyDetector:
    """
    AI-generated videos often show hand/finger anomalies:
    - Extra digits (6 fingers instead of 5)
    - Missing fingers
    - Fingers fused together or wrong proportions
    - Hands that clip through objects
    """
    
    def __init__(self):
        self.hand_model = self.load_hand_detector()  # MediaPipe Hands
    
    def detect_anomaly(self, frame):
        """Detect hand anomalies in a frame"""
        results = self.hand_model.process(cv2.cvtColor(frame, cv2.COLOR_BGR2RGB))
        
        anomaly_score = 0.0
        
        for hand_landmarks in results.multi_hand_landmarks:
            fingers = self.extract_finger_segments(hand_landmarks)
            
            # Check finger count
            finger_count = self.count_visible_fingers(fingers)
            if finger_count > 5:
                anomaly_score += 0.4  # Extra finger
            elif finger_count < 4 and len(fingers) > 0:
                anomaly_score += 0.3  # Missing finger
            
            # Check finger length ratios
            length_ratios = self.compute_finger_length_ratios(fingers)
            if self.are_ratios_anomalous(length_ratios):
                anomaly_score += 0.3
            
            # Check for fused fingers (too close landmark positions)
            if self.has_fused_fingers(hand_landmarks):
                anomaly_score += 0.3
        
        return min(anomaly_score, 1.0)
    
    def extract_finger_segments(self, hand_landmarks):
        """Extract finger bone segments from hand landmarks"""
        # Finger joint indices for MediaPipe hands
        finger_tips = [4, 8, 12, 16, 20]  # Thumb, Index, Middle, Ring, Pinky tips
        finger_pips = [3, 6, 10, 14, 18]  # PIP joints
        
        fingers = []
        for tip, pip in zip(finger_tips, finger_pips):
            tip_pos = hand_landmarks.landmark[tip]
            pip_pos = hand_landmarks.landmark[pip]
            fingers.append({
                'tip': (tip_pos.x, tip_pos.y, tip_pos.z),
                'pip': (pip_pos.x, pip_pos.y, pip_pos.z)
            })
        return fingers
    
    def count_visible_fingers(self, fingers):
        """Count how many fingers are extended"""
        # A finger is extended if tip is further from wrist than PIP
        extended_count = 0
        for finger in fingers:
            tip_dist = self.distance_to_wrist(finger['tip'])
            pip_dist = self.distance_to_wrist(finger['pip'])
            if tip_dist > pip_dist * 1.1:
                extended_count += 1
        return extended_count
    
    def are_ratios_anomalous(self, length_ratios):
        """
        Check if finger length ratios are biologically plausible.
        Normal ratio (Index/Middle ~0.95, Ring/Middle ~0.9, Pinky/Index ~0.7)
        """
        expected_ratios = {
            'index_to_middle': (0.88, 1.02),
            'ring_to_middle': (0.85, 0.98),
            'pinky_to_index': (0.62, 0.78)
        }
        
        for ratio_name, (min_r, max_r) in expected_ratios.items():
            if ratio_name in length_ratios:
                if not (min_r <= length_ratios[ratio_name] <= max_r):
                    return True
        return False
```

### H. Lighting & Shadow Inconsistencies

**Why it matters:** AI generators often fail to maintain consistent lighting across the entire frame. Shadows may not match the light source direction, or objects may have no shadow when they should.

#### Lighting Consistency Analysis
```python
class LightingAnalyzer:
    """
    Detect lighting/shadow inconsistencies in video frames.
    Based on Skyra research: Lighting Inconsistency found in ~2.5% of AI-generated videos.
    """
    
    def detect_anomaly(self, frame):
        """Analyze lighting consistency across the frame"""
        # Convert to grayscale for brightness analysis
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        # Divide frame into grid regions
        h, w = gray.shape
        grid_size = 8  # 8x8 grid
        cell_h, cell_w = h // grid_size, w // grid_size
        
        brightness_per_cell = []
        for i in range(grid_size):
            for j in range(grid_size):
                cell = gray[i*cell_h:(i+1)*cell_h, j*cell_w:(j+1)*cell_w]
                brightness_per_cell.append(np.mean(cell))
        
        # Real scenes: brightness varies smoothly with lighting
        # AI generated: abrupt brightness changes don't follow physics
        
        brightness_grid = np.array(brightness_per_cell).reshape(grid_size, grid_size)
        
        # Compute spatial gradient (smoothness of lighting)
        grad_x = np.abs(np.diff(brightness_grid, axis=1))
        grad_y = np.abs(np.diff(brightness_grid, axis=0))
        
        # Large gradients indicate unnatural lighting
        max_gradient = max(np.max(grad_x), np.max(grad_y))
        
        if max_gradient > LIGHTING_GRADIENT_THRESHOLD:
            return 0.5  # Suspicious
        
        return 0.0
    
    def detect_shadow_inconsistency(self, frame, face_landmarks):
        """
        Check if shadows on face are consistent with background lighting.
        """
        # Estimate light direction from background
        background_region = self.get_background_region(frame, face_landmarks)
        light_direction = self.estimate_light_direction(background_region)
        
        # Check face shadow direction
        face_shadow_dir = self.estimate_face_shadow_direction(face_landmarks)
        
        # If shadow direction doesn't match light direction
        angle_diff = self.angle_between(light_direction, face_shadow_dir)
        
        if angle_diff > SHADOW_ANGLE_THRESHOLD:  # > 30 degrees
            return 0.6  # Suspicious
        
        return 0.0
```

### I. Background & Object Permanence Artifacts

**Key concept:** Real videos maintain "object permanence" — objects continue to exist and move logically when partially occluded. AI videos often make objects appear/disappear incorrectly.

#### Temporal Background Consistency
```python
class BackgroundAnalyzer:
    """
    Detect background inconsistencies across frames.
    AI often fails to maintain consistent background representation.
    """
    
    def detect_object_appearance_artifacts(self, frames):
        """
        Track background elements across frames.
        Objects should persist or logically disappear/appear.
        """
        # Use background segmentation to extract static elements
        
        bg_model = self.load_background_segmentation_model()  # RMBG-1.4 or similar
        
        frame_backgrounds = []
        for frame in frames:
            mask = bg_model.predict(frame)
            bg = cv2.bitwise_and(frame, frame, mask=mask)
            frame_backgrounds.append(bg)
        
        # Compare background across frames
        # For each pixel position, track if background object unexpectedly changes
        
        inconsistencies = []
        for i in range(1, len(frame_backgrounds)):
            prev_bg = cv2.cvtColor(frame_backgrounds[i-1], cv2.COLOR_BGR2GRAY)
            curr_bg = cv2.cvtColor(frame_backgrounds[i], cv2.COLOR_BGR2GRAY)
            
            # Pixel-wise difference
            diff = cv2.absdiff(prev_bg, curr_bg)
            
            # Large diff in background region = object permanence violation
            large_diff_mask = diff > BACKGROUND_CHANGE_THRESHOLD
            
            if np.mean(large_diff_mask) > 0.05:  # >5% of background changed
                inconsistencies.append({
                    'frame': i,
                    'change_ratio': np.mean(large_diff_mask)
                })
        
        return inconsistencies
    
    def detect_occlusion_violations(self, frame, prev_frame, object_mask):
        """
        When an object moves, its occlusion relationship with background should be preserved.
        AI sometimes shows objects "phase through" or incorrectly occlude background elements.
        """
        # Compute optical flow for background pixels
        prev_gray = cv2.cvtColor(prev_frame, cv2.COLOR_BGR2GRAY)
        curr_gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        flow = cv2.calcOpticalFlowFarneback(
            prev_gray, curr_gray, None,
            pyr_scale=0.5, levels=3, winsize=15,
            iterations=3, poly_n=5, poly_sigma=1.2
        )
        
        # Check flow consistency in background regions (not object pixels)
        background_flow = flow * (1 - object_mask)[..., np.newaxis]
        
        # Flow should be coherent in background (no sudden jumps)
        flow_magnitude = np.linalg.norm(background_flow, axis=2)
        
        # Sudden flow spikes indicate occlusion violation
        flow_spike_mask = flow_magnitude > FLOW_SPIKE_THRESHOLD
        
        if np.mean(flow_spike_mask) > 0.02:  # >2% of background has flow spikes
            return 0.5
        
        return 0.0
```

### J. Audio-Visual Sync & Prosody Analysis

**Why it matters:** AI-generated talking head videos often have subtle audio-visual mismatches beyond just lip sync — the prosody (rhythm, emphasis, emotion) may not match the facial expression.

#### Audio-Visual Consistency Detector

    """
    Detect audio-visual inconsistencies beyond lip sync.
    Based on affective coherence analysis (Adeia research).
    """

                break

            expr = self.extract_facial_expression(frame)
            frame_expressions.append(expr)

        }

        # AU intensities (approximated via landmark geometry)
        
        # AU1: Inner brow raiser
        # AU2: Outer brow raiser  
        # AU4: Brow lowerer
        # AU6: Cheek raiser
        # AU12: Lip corner pull
        # etc.
        
        # For simplicity, compute basic emotion proxy
        mouth_height = self.landmark_distance(landmarks[14], landmarks[13])  # Open mouth

### K. Identity Consistency (TI2Net-style)

**Key concept:** In real videos, a person's identity (face features, voice, mannerisms) stays consistent. Deepfakes can show "temporal identity inconsistency" where the same person looks slightly different across frames.

#### Temporal Identity Consistency Checker
```python
class IdentityConsistencyChecker:
    """
    Based on TI2Net (Temporal Identity Inconsistency Network) - WACV 2023.
    Detects when face identity shifts across frames in deepfake videos.
    """
    
    def compute_identity_embedding(self, face_roi):
        """Extract face embedding for identity representation"""
        # Use pre-trained face recognition model (FaceNet, ArcFace, etc.)
        embedding = self.face_model.extract_embedding(face_roi)
        return embedding
    
    def detect_identity_drift(self, frames_with_faces):
        """
        Check if face identity stays consistent across video.
        Deepfakes can subtly alter face appearance frame-to-frame.
        """
        embeddings = []
        timestamps = []
        
        for frame, ts in frames_with_faces:
            face_roi = self.extract_face_roi(frame)
            embedding = self.compute_identity_embedding(face_roi)
            embeddings.append(embedding)
            timestamps.append(ts)
        
        # Compute pairwise identity similarity
        # In real video: same person = high similarity across all frames
        # In deepfake: identity similarity fluctuates
        
        inconsistencies = []
        for i in range(len(embeddings)):
            for j in range(i+1, len(embeddings)):
                similarity = cosine_similarity(embeddings[i], embeddings[j])
                
                # If same person but similarity drops significantly
                if similarity < IDENTITY_SIMILARITY_THRESHOLD:
                    time_diff = abs(timestamps[i] - timestamps[j])
                    
                    # Close frames should have similar identity
                    if time_diff < 1.0:  # Within 1 second
                        inconsistencies.append({
                            'frames': (i, j),
                            'similarity': similarity,
                            'time_diff': time_diff
                        })
        
        # Calculate identity drift score
        if len(inconsistencies) > 0:
            avg_similarity = np.mean([x['similarity'] for x in inconsistencies])
            return 1.0 - avg_similarity  # Lower similarity = higher anomaly
        
        return 0.0
```

### L. Compression & Re-encoding Artifacts

**Key insight:** YouTube re-encodes all videos, which strips metadata and can degrade some detection signals. However, re-encoding also creates its own artifacts that can be detected.

#### Detect Re-encoding Artifacts
```python
class CompressionArtifactDetector:
    """
    Detect signs of heavy re-encoding (YouTube, etc.) which may indicate
    attempt to hide AI generation artifacts.
    """
    
    def detect_double_compression(self, frame):
        """
        When AI-generated video is re-uploaded, double compression creates
        specific artifacts in DCT domain.
        """
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        # 2D DCT
        dct = cv2.dct(gray.astype(float))
        
        # Check for quantization artifacts in DCT coefficients
        # Double compression creates periodic patterns in DCT domain
        
        # Analyze high-frequency DCT coefficients
        h, w = dct.shape
        hf_energy = np.mean(np.abs(dct[h//2:, w//2:]))
        lf_energy = np.mean(np.abs(dct[:h//4, :w//4]))
        
        # Unusually low HF energy = heavy compression
        compression_ratio = hf_energy / (lf_energy + 1e-6)
        
        if compression_ratio < COMPRESSION_THRESHOLD:
            return {
                'score': 0.4,
                'type': 'heavy_compression',
                'note': 'May hide AI artifacts'
            }
        
        return {'score': 0.0}
    
    def estimate_quality(self, frame):
        """
        Estimate video quality (low quality often used to hide AI artifacts).
        """
        gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        
        # BRISQUE-like features (Blind/Referenceless Image Spatial Quality Evaluator)
        # Compute natural scene statistics
        
        # Local variance normalization
        local_mean = cv2.blur(gray, ksize=15)
        local_var = cv2.blur((gray - local_mean)**2, ksize=15)
        
        # Quality score based on variance statistics
        # Lower variance = lower quality
        mean_var = np.mean(local_var)
        
        if mean_var < LOW_QUALITY_VARIANCE_THRESHOLD:
            return 0.5  # Low quality
        
        return 0.0
```
